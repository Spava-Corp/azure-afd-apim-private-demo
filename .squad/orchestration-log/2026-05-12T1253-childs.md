## Summary

Hardened NSG ruleset for ESLZ compliance and added missing private DNS zone for AKS private cluster API server resolution.

## Changes

### 1. Explicit Deny-All Rules (Priority 4000)

Added to all four NSGs (APIM, AKS, PE, Bastion):
- `Deny-All-Inbound` (P4000, action: Deny, source: *)
- `Deny-All-Outbound` (P4000, action: Deny, destination: *)

**Rationale:** ESLZ policy audits expect explicit deny rules below priority 4096 for visibility, even though Azure implicit deny exists at 65500.

### 2. APIM HTTPS Inbound Rule Tightening

**Rule:** `Allow-HTTPS-Inbound` (Priority 120) on `apimNsg`
- **Changed:** Source from `VirtualNetwork` to `10.1.6.0/24` (PE subnet only)
- **Impact:** Only AFD Private Endpoint can reach APIM on port 443; prevents lateral movement from AKS/Bastion

### 3. Private Endpoints Subnet NSG Populated

**New rule:** `Allow-HTTPS-From-VNet` (Priority 100)
- **Direction:** Inbound
- **Port:** 443
- **Source:** `VirtualNetwork`
- **Impact:** Minimum required for private endpoint traffic from APIM and other resources

### 4. AKS Outbound Allowlist

Added three essential outbound rules to `aksNsg`:
- **P110 — ACR:** Destination ServiceTag `AzureContainerRegistry`, port 443
  - Purpose: Pull container images
- **P120 — AAD:** Destination ServiceTag `AzureActiveDirectory`, port 443
  - Purpose: Control plane auth, managed identity token acquisition
- **P130 — AzureMonitor:** Destination ServiceTag `AzureMonitor`, port 443
  - Purpose: Container Insights, log shipping, metrics
- **P4000 — Deny All Outbound:** Explicit deny fallback

**Impact:** AKS cluster is now network-restrictive. Workload egress to external APIs/services requires additional rules.
